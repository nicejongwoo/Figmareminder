# 테스트 계획서

## 테스트 개요

이 문서는 상황 기반 지능형 리마인더 앱의 테스트 항목을 정의합니다.
각 기능별로 수동 테스트 시나리오를 제공하며, 향후 자동화 테스트 작성 시 참고 자료로 활용됩니다.

## 1. 리마인더 생성/수정

### 1.1 기본 정보 입력

#### TC-001: 제목만 입력하여 생성
- **전제조건**: 홈 화면
- **단계**:
  1. 하단 중앙 '+' 버튼 클릭
  2. 제목에 "테스트 리마인더" 입력
  3. '저장' 버튼 클릭
- **예상 결과**:
  - 리마인더가 생성됨
  - 토스트 알림 표시: "✅ 리마인더 생성됨"
  - 홈 화면에 새 리마인더 표시

#### TC-002: 모든 필드 입력하여 생성
- **전제조건**: 홈 화면
- **단계**:
  1. '+' 버튼 클릭
  2. 제목: "완전한 리마인더"
  3. 설명: "상세 설명 텍스트"
  4. 아이콘: 📱 선택
  5. 우선순위: 긴급
  6. 그룹: 외출 전
  7. '저장' 버튼 클릭
- **예상 결과**:
  - 리마인더가 긴급 섹션에 생성됨
  - 그룹 배지 표시: "외출 전"

#### TC-003: 빈 제목으로 생성 시도
- **전제조건**: 리마인더 생성 다이얼로그
- **단계**:
  1. 제목을 비워둠
  2. '저장' 버튼 클릭
- **예상 결과**:
  - 에러 메시지 또는 필드 강조
  - 리마인더가 생성되지 않음

### 1.2 트리거 설정

#### TC-004: 시간 기반 트리거
- **전제조건**: 리마인더 생성 다이얼로그
- **단계**:
  1. 트리거: "시간" 선택
  2. 시간: "14:30" 입력
  3. 요일: 월, 수, 금 선택
  4. 저장
- **예상 결과**:
  - 리마인더에 "⏰ 월, 수, 금 오후 2:30" 표시

#### TC-005: 위치 기반 트리거
- **전제조건**: 리마인더 생성 다이얼로그
- **단계**:
  1. 트리거: "위치" 선택
  2. 위치: "우리 집" 선택
  3. 트리거 타입: "떠날 때"
  4. 저장
- **예상 결과**:
  - 리마인더에 "📍 우리 집 (떠날 때)" 표시
  - 홈 화면에 위치 추적 상태 표시

#### TC-006: 복합 트리거 (시간 + 위치)
- **전제조건**: 리마인더 생성 다이얼로그
- **단계**:
  1. 트리거: "시간과 위치 모두" 선택
  2. 시간: "09:00"
  3. 위치: "회사"
  4. 저장
- **예상 결과**:
  - 리마인더에 시간과 위치 정보 모두 표시

### 1.3 체크리스트

#### TC-007: 체크리스트 항목 추가
- **전제조건**: 리마인더 생성 다이얼로그
- **단계**:
  1. 체크리스트 입력 필드에 "항목 1" 입력
  2. Enter 또는 '+' 버튼
  3. "항목 2" 추가
  4. "항목 3" 추가
  5. 저장
- **예상 결과**:
  - 리마인더 카드에 "0/3" 표시
  - 상세 보기에서 3개 항목 표시

#### TC-008: 체크리스트 항목 삭제
- **전제조건**: 체크리스트 항목이 있는 상태
- **단계**:
  1. 항목 옆 '×' 버튼 클릭
- **예상 결과**:
  - 항목이 삭제됨
  - 개수 업데이트

#### TC-009: 빈 체크리스트 항목 추가 시도
- **전제조건**: 체크리스트 입력 필드
- **단계**:
  1. 빈 상태에서 '+' 버튼 클릭
- **예상 결과**:
  - 항목이 추가되지 않음

### 1.4 리마인더 수정

#### TC-010: 기존 리마인더 수정
- **전제조건**: 리마인더가 최소 1개 존재
- **단계**:
  1. 리마인더 상세 보기 열기
  2. '편집' 버튼 클릭
  3. 제목을 "수정된 제목"으로 변경
  4. 저장
- **예상 결과**:
  - 토스트: "✅ 리마인더 수정됨"
  - 변경사항이 즉시 반영됨

## 2. 체크리스트 완료

### 2.1 항목 완료

#### TC-011: 단일 항목 체크
- **전제조건**: 체크리스트가 있는 리마인더
- **단계**:
  1. 체크박스 클릭
- **예상 결과**:
  - 체크 표시됨
  - 텍스트에 취소선
  - 완료율 업데이트 (예: 0/3 → 1/3)
  - 프로그레스 바 업데이트

#### TC-012: 항목 체크 해제
- **전제조건**: 체크된 항목
- **단계**:
  1. 체크박스 다시 클릭
- **예상 결과**:
  - 체크 해제됨
  - 취소선 제거
  - 완료율 감소

#### TC-013: 전체 완료
- **전제조건**: 3개 항목, 2개 완료됨
- **단계**:
  1. 마지막 항목 체크
- **예상 결과**:
  - 토스트: "🎉 리마인더 완료!"
  - `totalCompletions` 증가
  - `lastCompleted` 업데이트
  - 통계에 반영

#### TC-014: 재완료 (전체 체크 해제 후 다시 완료)
- **전제조건**: 전체 완료된 리마인더
- **단계**:
  1. 한 항목 체크 해제
  2. 다시 체크
- **예상 결과**:
  - 다시 완료 토스트 표시
  - 통계 다시 증가
